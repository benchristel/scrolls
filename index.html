Writing your first JavaScript program

1. Get a code editor

If you want to write more than a few lines of JavaScript, you're going to need software that's designed for editing code files. Word processors (Microsoft Word or Apple Pages) won't work for writing code, because they can't save files in the right format. Bare-bones text editors like TextEdit are okay, but lack useful features like syntax checking.

Komodo Edit is a good code editor that's available for free for Mac, Windows, and Linux. I recommend downloading it and using it for the projects in this book.

2. Download the example file

Once you've done that, download this example file, which contains everything you need to start haxxing teh codez. Open it in both your editor and web browser (Command-O opens files in both Chrome and Komodo Edit).

3. Test out the example in the browser

When you open the example file in the browser, you'll just see a blank page. Let's open up the JavaScript Console (by pressing option+command+J) and test out a few commands.

The example file includes Thrax, a set of tools for creating user interfaces. Try out this command in the console:

Thrax.createButton();

You'll see the button appear in the top-left corner of the screen. Unfortunately, we can't make this button do anything now because we didn't give it a name when we created it. If something in JavaScript doesn't have a name, there's no way we can refer to it, and thus no way to control it. Let's start over.

Thrax.clearScreen();

Now we can create another button and give it a name.

var myButton = Thrax.createButton();

I've chosen to call this button myButton, but the name is arbitrary; we could have called it b or fooBar or qwer_1234. A name (or identifier) in JavaScript can contain letters, numbers, underscores (_) and dollar signs, so if you wanted to, you could name your button O_o or $$$.
